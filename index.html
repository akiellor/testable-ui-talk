The Problem

We wanted a friendly UI
that we could change with confidence

Our Options

  • Selenium, Webdriver on a real browser (slow)
  • Manually checking interactions (error-prone)
  • Headless, browser emulators like envjs and htmlunit (not quite the real thing)
  • Headless browsers like phantomjs (interesting)
  • Decoupling from the DOM and using the MVVM pattern (interesting)

We Brought in KnockoutJS -- here are the pros

It's implementation of the MVVM pattern meant we could just right plain ol' javascript objects, which are easy to unit-test and don't require testing the integration with the DOM. Knockout's job is binding to the dom, which we could visually verify was working correctly.

It also meant we could use Mozilla's JS Engine Rhino for testing, since we were free of the DOM
