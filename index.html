Problem

We wanted to build a friendly* UI
that we could change with confidence




* One that encourages success, and prevents obvious failures

Options

  • Manual
  • Selenium driving Firefox
  • Headless browsers like PhantomJS
  • Decoupling from the DOM using the Presentation Model* pattern (yes!)

* aka MVVM

Presentation Model with KnockoutJS

  • Focused tests
  • Leave the DOM out of it* — we know it works!
  • Fast tests — behavior tested headlessly in a JS runtime


* a few tests through the whole system worked for us
